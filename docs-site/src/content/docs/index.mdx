---
title: Keepsake
description: Deterministic relation lifecycles for Rust applications.
---

Keepsake stores relation lifecycles for Rust applications: tags, sanctions,
entitlements, holds, risk flags, feature gates, and other cases where a subject
has a relation until policy changes.

It gives you deterministic writes, expiry, reads, worker scans, and typed audit
records. Users, accounts, authorization, tenancy, display data, and
product-specific joins stay where they belong, in your application.

Start with the overview if you are new to the model. Use the quickstart when
you want to apply one relation in a Rust/Postgres service. Use the guides for
specific patterns, and the reference pages when you need exact API contracts.
