---
title: Versioning
description: Crate, API, and schema compatibility expectations.
---

Keepsake uses crate versions for both API and schema expectations. Before 1.0,
minor versions may still refine public contracts. Check the changelog before
upgrading, especially when `keepsake-sqlx` adds or changes migrations.

## Upgrade Checklist

- Pin `keepsake` and `keepsake-sqlx` together.
- Read the changelog for API changes, new migration files, changed indexes, and
  required ordering.
- Apply matching migrations before deploying code that depends on new schema.
- Test request paths and workers that use changed query helpers.

Embedded migrations define the required schema. Your service still decides when
and how to apply it.
