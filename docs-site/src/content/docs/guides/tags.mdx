---
title: Tags
description: Model durable tags with manual expiry.
---

Tags are the smallest Keepsake integration. They represent durable membership:
an account is trusted, a user has beta access, or an organization belongs to a
review cohort until an application command removes that relation.

Use `ExpiryPolicy::ManualOnly` for tags:

```rust
use keepsake::ExpiryPolicy;

keepsake::relation_spec! {
    struct TrustedTag {
        id: 0x018f_0000_0000_7000_8000_0000_0000_0001;
        key: ("tag", "trusted");
        expiry(_at) => ExpiryPolicy::ManualOnly;
    }
}
```

Apply the tag through a timestamp-bound repository:

```rust
use keepsake::{ActorRef, ApplyKeepsake, CommandContext};

let now = chrono::Utc::now();
let timed_repo = repo.at(now);

timed_repo.upsert_relation_spec::<TrustedTag>().await?;

let command = ApplyKeepsake::for_spec::<TrustedTag>(
    subject,
    now,
    CommandContext::new(ActorRef::new("system", "worker")?),
);
repo.apply(&command).await?;
```

After the apply succeeds, request paths can read active tags by subject. Batch
jobs can scan active members by relation id when they need every subject with
that tag.

Tags are the simplest integration because they do not require background expiry
workers or fulfillment projections. The application defines what the tag means,
who may apply or revoke it, and how tag rows join back to user or account
tables.
